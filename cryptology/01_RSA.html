    # 质数分解难题

    RSA 算法的安全性依赖于大整数进行质因数分解的困难

    # 前置知识

    # 质数

    1. 定义

      质数也称素数，它是指只能被 111 和本身整除的自然数

    2. 互质

      若两个数的最大公因数（或最大公约数）是 111，则称这两个数互质

    # 同余式

    这个概念很重要，对后面的 RSA 算法正确性的证明及其关键

    1. 定义

      若两个数 a、ba、ba、b 同时除以某一个数 mmm 的余数相同，则称这两个数 a、ba、ba、b 对某一个数 mmm 同余，记为 a≡b(modm)a \equiv b (mod m)a≡b(modm)，比如

      23÷7=3…223÷7=3 \dots 223÷7=3…2

      65÷7=9…265÷7=9 \dots 265÷7=9…2

      则 232323 和 656565 对 777 同余，记做 23≡65(mod7)23 \equiv 65 (mod 7)23≡65(mod7)

    2. 性质

      1. 同余式可以互相加：若 a≡b（modm）a ≡ b（mod m）a≡b（modm）、 c≡d（modm）c ≡ d（mod m）c≡d（modm），则 a+c≡b+d（modm）a+c ≡ b+d（mod m）a+c≡b+d（modm）

      2. 同余式可以互相乘：若 a≡b（modm）a ≡ b（mod m）a≡b（modm）、 c≡d（modm）c ≡ d（mod m）c≡d（modm），则 a∗c≡b∗d（modm）a*c ≡ b*d（mod m）a∗c≡b∗d（modm）

    # 欧拉函数

    1. 定义

      记欧拉函数 φ(n)\varphi (n)φ(n)，φ(n)\varphi (n)φ(n) 表示为小于 nnn 的正整数中与 nnn 互质的数的数目

    2. 如果 nnn 能写作两个不同质数 p1p_{1}p1​ 与 p2p_{2}p2​ 的乘积，即 n=p1×p2n=p_{1} \times p_{2}n=p1​×p2​，则 φ(n)=(p1−1)×(p2−1)\varphi(n) =(p_{1} - 1) \times (p_{2} - 1)φ(n)=(p1​−1)×(p2​−1)

      因为 φ(n)=φ(p1×p2)=φ(p1)×φ(p2)=(p1−1)×(p2−1)\varphi(n) = \varphi(p_{1} \times p_{2}) = \varphi(p_{1}) \times \varphi(p_{2}) = (p_{1} - 1) \times (p_{2} - 1)φ(n)=φ(p1​×p2​)=φ(p1​)×φ(p2​)=(p1​−1)×(p2​−1)

    # 欧拉定理

    若 aaa 与 nnn 互质，则 aφ(n)≡1(modn)a^{\varphi (n)} \equiv 1 ( mod n )aφ(n)≡1(modn)

    # 乘法逆元

    如果 ab≡1（modm）ab ≡ 1（mod m）ab≡1（modm），则称 aaa 和 bbb 为关于 mmm 互为乘法逆元

    已知 aaa 求 bbb 的方法：因为 ab≡1（modm）ab ≡ 1（mod m）ab≡1（modm），所以不妨设 ab=mk+1ab=mk+1ab=mk+1，其中 aaa 和 mmm 为已知数

    利用 扩展欧几里得 算法，可以在多项式时间内，计算出来一个乘法逆元 bbb，当然 kkk 也是能求出来的，只不过此处的 kkk 没啥用

    此处利用 扩展欧几里得算法 的前提是 aaa 与 mmm 互素

    # RSA 算法

    # 步骤

    1. 第一步：随机生成两个大质数 p1p_{1}p1​ 和 p2p_{2}p2​

    2. 第二步：计算 n=p1×p2n = p_{1} \times p_{2}n=p1​×p2​

    3. 第三步：计算欧拉函数 φ(n)=(p1−1)×(p2−1)\varphi (n) = (p_{1} - 1) \times (p_{2} - 1)φ(n)=(p1​−1)×(p2​−1)

    4. 第四步：构造一个比 111 大、比 φ(n)\varphi (n)φ(n) 小、不等于 p1p_{1}p1​ 或 p2p_{2}p2​ 的整数 eee

    5. 第五步：求出 eee 对于 φ(n)\varphi (n)φ(n) 的乘法逆元 ddd，也就是说 ed≡1（modφ(n)）ed \equiv 1（mod  \varphi(n)）ed≡1（modφ(n)），也就是说 ed=kφ(n)+1ed=k\varphi (n)+1ed=kφ(n)+1

    至此，公钥和私钥生成完毕，其中

    公钥： (n,e)(n, e)(n,e)

    私钥： (n,d)(n, d)(n,d)

    对于一个明文数据 mmm，其中 mmm 与 nnn 是互质的，并且 mmm 小于 nnn

    可以利用公钥 (n,e)(n, e)(n,e) 加密 mmm 为密文 ccc，具体加密算法如下

    c≡me(modn)\mathbf{\color{#f40} c \equiv m^{e} (mod n)}c≡me(modn)

    对于密文数据 ccc，可以利用私钥 (n,d)(n, d)(n,d) 解密 ccc 为明文 aaa，具体解密算法如下

    m≡cd(modn)\mathbf{\color{#008c8c} m \equiv c^{d} (mod n)}m≡cd(modn)

    # 证明

    以下的证明是基于明文数据 mmm 与 nnn 是互素的情况

    当 mmm 与 nnn 不是互素的话，结果也是一样的

    由于 mmm 与 nnn 互素，根据欧拉定理可知 aφ(n)≡1(modn)a^{\varphi (n)} \equiv 1 ( mod n )aφ(n)≡1(modn)，所以

    所以，由 (1)\color{#f40} (1)(1) 和 (2)\color{#f40} (2)(2) 式可知 aaa 与 cdc^{d}cd 模 nnn 的结果是一样的，即

    m≡cd(modn)\mathbf{\color{#008c8c} m \equiv c^{d} (mod n)}m≡cd(modn)

    为何 ddd 很难破解？

    攻击者可以获得的数据有密文数据 ccc 和公钥 (n,e)(n,e)(n,e)

    又因为

    m≡cd(modn)\color{orage} m \equiv c^{d} (mod n)m≡cd(modn)

    要想根据密文数据 ccc 获得明文数据 mmm，就必须再知道私钥 (n,d)(n,d)(n,d) 中的 ddd

    由于

    ed=kφ(n)+1ed=k\varphi (n)+1 ed=kφ(n)+1

    所以只要知道 φ(n)\varphi (n)φ(n)，就能求出 ddd

    又因为

    φ(n)=(p1−1)(p2−1)\varphi (n) = (p_{1}-1)(p_{2}-1)φ(n)=(p1​−1)(p2​−1)

    所以只要知道 p1 和 p2p_{1} 和 p_{2}p1​和p2​ 就能求出 φ(n)\varphi (n)φ(n) 继而求出 ddd

    又因为

    n=p1×p2n = p_{1} \times p_{2}n=p1​×p2​

    所以只要通过 nnn 求得 p1 和 p2p_{1} 和 p_{2}p1​和p2​ 即可求出 ddd

    但是因为 nnn 太大了，而分解质因数需要指数时间，所以没有人能够将 nnn 质因数分解求出 p1p_{1}p1​ 和 p2p_{2}p2​

    这种将大整数进行质因数分解的问题就称为质数分解难题

    所以理论上任何人都不能推导出私钥 ddd

    RSA 的安全性条件

    1. ∣p−q∣|p-q|∣p−q∣ 要大

    2. p−1p-1p−1 和 q−1q-1q−1 都应有大素因子
