# 导读

我们通过 new Constructor() 这种形式来创建对象应该非常熟悉了，但是 new 关键字内部究竟发生了什么，你是否有探究过呢？接下来让我们一步步揭开它神秘的面纱吧

# 一、构造函数的内部原理(隐式三段论)

假设我们写了一个这样的构造函数：function Person(){}，系统会为 Person 构造函数添加一个静态属性 prototype，该属性为一个对象，该对象有两个属性，分别为 constructor 和 __proto__，即

function Person(){};
//下面代码是系统自动添加的
Person.prototype = {
  constructor: Person,//指向Person构造函数
  __proto__: Object//指向Object对象
}

我们通过 new 关键字来创建一个对象 person ，如 var person = new Persopn(); 时，构造函数内部会发生以下三步：

  1. 在函数体第一行系统开始添加如下代码（系统内部做的操作我们是看不见的）



     
     
     


    function Person(){
  //1.系统添加如下代码
  var this = {
    __proto__ : Person.prototype,
  };
}

  2. 增属性(自己手动添加的)






     


    function Person(){
  //1.系统添加如下代码
  var this = {
    __proto__ : Person.prototype,
  };
  //2.我们手动添加的代码...，如 this.age=18
}

  3. 系统会隐式返回 this








     
     


    function Person(){
  //1.系统添加如下代码
  var this = {
    __proto__ : Person.prototype,
  };
  //2.我们手动添加的代码...，如this.age=18

  //3. 系统添加如下代码
  return this;
}

# 二、别样的礼物

根据上面的内容我们已经了解了构造函数内部的 this 原理了，但是接下来我们再进一步探讨构在造函数体内我们若是显示地返回一个数据呢，构造函数是否仍然会返回 this ？

  1. 显示返回一个原始值
function Test(){
  this.name = "lkj";
  this.age = 12;
  return 12;
}
var test = new Test();
console.log(test);

  1. 显示返回一个对象
function Test(){
  var obj = {
    number: 2299999,
    sex: "male"
  }
  this.name = "lkj";
  this.age = 12;
  return obj;
}
var test = new Test();
console.log(test);

「总结」

若你在构造函数中显示返回一个对象(如函数 数组等) 那么隐式第三步（return this）就不会执行；若你在构造函数中返回原始值，隐式第三步（return this）仍会执行（会忽略掉返回的原始值）
